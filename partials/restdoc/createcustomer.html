How to create a customer


Creating a customer is pretty simple, since there is only one value which is mandatory.

	{
	  "email": "jane@doe.com",
	  "merchantCustomerId": "xyz"
	}


Create Customer Body - JSON Format
# Parameter Mandatory Description
1 email mandatory E-Mail address of the customer
2 merchantCustomerId optional Optional merchant side id for the customer

With the body ready we can now send the request via curl.

	curl --user MerchantID:APIKey -H "Content-Type: application/json" -X POST -d '{"email":"jane@doe.com","merchantCustomerId":"xyz"}' https://apitest.payengine.de/v1/customers


We will now touch on every part of this command.

--user MerchantID:APIKey
Curl Option for HTTP basic access authentication.

-H "Content-Type: application/json"> content-type header "application/json" for our API.
-X POST
We want the request to be a HTTP POST.

-d '{"email":"jane@doe.com","merchantCustomerId":"xyz"}'
The body for our request in JSON format.

https://apitest.payengine.de/v1/customers
The customers endpoint of our API (Test in this case).

If the Request was successful we should get a response that looks like this:

	{
	  "createdAt": 1501757581860,
	  "modifiedAt": 1501757581860,
	  "email": "jane@doe.com",
	  "merchantCustomerId": "xyz",
	  "customerId": "Customer-xxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxx",
	  "addresses": [],
	  "personas": [],
	  "paymentInstruments": []
	}


Customer Response Body - JSON Format
# Parameter Description
1 createdAt Unix timestamp of the creation of the customer
2 modifiedAt Unix timestamp of last modification of the customer
3 email The email address set for the customer. Should be the same as in the request.
4 merchantCustomerId The customer ID you defined. Should be the same as in the request.
5 customerId The Payengine customer id, important for further requests.
6 addresses Addresses associated with the customer.
7 personas Personas associated with the customer.
8 paymentInstruments Payment Instruments associated with the customer.
