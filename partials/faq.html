Transaction status


After creating a debit order with the payment method Sofort the transaction status will be PENDING and a redirect URL will be provided.
The end customer needs to be redirected to the redirect URL.
When the end customer completes his payment via Sofort, the transaction status results either in SUCCESS or in case of failure in FAILURE.

With Deutsche Handelsbank bank account


Payment loss


In case that the payment does not arrive to your bank account, a new chargeback transaction in the status CHARGEBACK will be created as a child of the initial debit transaction. In this case you should directly contact the end customer to inform him that there is an issue with the payment.
The end customer can initiate a bank transfer with the order.paymentProviderTransactionId as reference to complete the transaction.
You can find the order.paymentProviderTransactionId in the Merchant Center detail view of your order.
To read the order information via API click here.

Payment received


As soon as the payment amount is available on your bank account, the flag order.settled will be set to true. If there was a chargeback transaction present, the chargeback status will change from CHARGEBACK to BOOKIN. See API spec for reading order information here.

Without Deutsche Handelsbank bank account


No further status updates will be available.

Support


If you need to contact the Sofort support to solve a transaction specific problem, ensure that you have always provided the order.paymentProviderTransactionId.

A decline can have multiple reasons. In case a transaction got declined because of customer or payment instrument details, there will be a dedicated exception message within Payengines API response. The latter can also be found within transaction details in Merchant Center.

Each individual card, bank or user account that is used for payment within Payengine is transformed into a ‚Payment Instrument‘ whose unique ID represents an encrypted token. Recurring payments can be initiated by using these paymentInstrumentId’s when communicating with Payengine API or within Merchant Center, provided the according payment method or scheme is supporting recurring payments at all. Please note that a credit card must first be successfully authenticated through strong authentication process such as 3D Secure before it supports ‚recurring‘.

Payment methods for which Concardis is acting as Acquirer (amongst others cards, paydirekt, SEPA and Giropay) will be settled and payed out according to the agreement within Payengine service contract. Please check for the settlement interval and a possible payout delay in order, both being part of the Payengine service contract. All other payment methods will be settled and payed out by the schemes directly according to the individual contract that you and the scheme agreed upon.

Most banks only transfer funds during working days. Bank holidays or weekends may delay the transfer of your funds.

Orders and transactions always have a dedicated status that represents the exact state of a customer payment. A successful payment is indicated by the debit or capture transaction state ‚SUCCESS‘. A detailed list of all status including the state transition rules can be found here

You should initiate the shipping only if the according capture or debit transaction is in status ‚SUCCESS‘. If you ship at an earlier stage you might risk a financial loss, since the transaction could still end in one of the negative and final status ‚FAILURE‘, ‚DECLINED‘, ‚ABORTED‘ or ‚EXPIRED‘.
