Transaction status


After creating a debit order with the payment method Sofort the transaction status will be PENDING and a redirect URL will be provided.
The end customer needs to be redirected to the redirect URL.
When the end customer completes his payment via Sofort, the transaction status results either in SUCCESS or in case of failure in FAILURE.

With Deutsche Handelsbank bank account


Payment loss


In case that the payment does not arrive to your bank account, a new chargeback transaction in the status CHARGEBACK will be created as a child of the initial debit transaction. In this case you should directly contact the end customer to inform him that there is an issue with the payment.
The end customer can initiate a bank transfer with the order.paymentProviderTransactionId as reference to complete the transaction.
You can find the order.paymentProviderTransactionId in the Merchant Center detail view of your order.
To read the order information via API click here.

Payment received


As soon as the payment amount is available on your bank account, the flag order.settled will be set to true. If there was a chargeback transaction present, the chargeback status will change from CHARGEBACK to BOOKIN. See API spec for reading order information here.

Without Deutsche Handelsbank bank account


No further status updates will be available.

SEPA Direct Debit

The Single Euro Payments Area (SEPA) is an EU payment-integration initiative to simplify EUR-based bank-to bank transactions. SEPA Direct Debit (SDD) is the Direct Debit scheme implemented for the Eurozone counties, allowing creditors to collect (pull) funds from their debtors' bank accounts.

Transactions


BookingRecords is created for every debit or capture. When merchant create an order that creates а BookingRecord. The direct debits are dispatched at configurable intervals.

All not processed BookingRecords will be processed in a configurable interval. Processing file contain transactions to debit the end customer and the merchants. - end customer accounts is debited with the amount from debit or capture transactions. - merchant accounts is debited in case of returning direct debits (RTI) with the RTI fee and if already transferred to the merchant, the returned direct debit amount.

After creating a debit/preauth order with the payment method SEPA DD the transaction status will be "OK" and the end customer will be redirected to the success redirect URL.

To keep the platoform in sync with all trust account bookings, we pull CAMT files in a configured interval. Notifications is send for incoming funds and for returned direct debits (chargebacks). If we receive a returned direct debit while trying to debit the merchant with RTI fees, we disable SEPA direct debit as payment methods and informs the merchant the he will need to contact the support.


We perform Merchant payouts in a configurable interval, the payout delay can be configured for every merchant individually. The merchant payout is processed as a SEPA transfer.


Recurring


As soon as a SEPA payment instrument is created it can be used in a recurring manner. A recurring payment request using the SEPA Direct Debit option is similar to a standard authorise payment request, except that recurring flag is set. For some European payment methods that do not support recurring, we have implemented a workaround that uses SEPA recurring payments for subsequent transactions.
