Navigation

Notifications can be found in "Notifications" menu item.



Create new notification

  • Click on „Plus“ icon on top right corner
    •  
  • New window will open to create a notification
    •  
  • Select notification type. (In this Example we will use Email notification)
    •   Note: There are two different notification types you can use( See more... ):
    • webhooks (URL) - deliver event information directly to your system by calling a script hosted on your server
    • Email - event information is sent to the specified email address
     
  • A new email field will appear
    •  
  • Fill in the notification information
    •   Note:
    • E-mail (respectively URL field, when URL notification type is selected) is required
    • Name and description are optional.
     
  • Select the events for which you want to receive notifications ( See more about Notification events... )
    •  

    You can select one or more events when you create a new notification. If you want to select all avaliable events, you can use the "Select All" option.

     
  • Activate Notification
    • Important:

    By default, the "Activate notification" box is selected when you create a new notification. So, if you do not want to activate it, you must uncheck it explicitly!

     
  • Save new Notification by selecting "Save" button on the bottom of the page
    •  
  • You can now see the created new notification in the notification list.

Delete Notification

To delete a notification, you must click the delete icon of the selected notification.

You will see modal window where you must confirm that you want to delete notification.

After confirmation,the selected notification will be deleted from notifications lists.

Notification Types

There are two different notification types a merchant can use: Email and URL notifications.

  • webhooks (URL) - deliver event information directly to your system by calling a script hosted on your server

    • Format: Webhooks are sent as HTTP POST requests on the given URI. Accepted are ports 80,8080 as well 443 for HTTPS.

    Notes:

    • A valid SSL certificate is needed in order to receive webhooks via HTTPS, which is highly recommended.
    • Basic authentication is possible through standard ( and already deprecated) url encoding: https://Aladdin:OpenSesame@www.example.com/
    • The event is sent in the request body as application/json.

    The message format of the events is as following:

    { createdAt: unix timestamp. ( contained in the event ) eventId: the unique identifier for this event. ( contained in the event ) notificationApiVersion: the version of the notification API, currently only "1.0" apiVersion: the API version that triggered the event ( contained in the event ) ipAddress: the ip address that caused the request ( contained in the event ) globalTrackId: either a string or null. corresponds to the ID of the request that cause the event. ( contained in the event ) mode: ‘live’ or ‘test’ type: event type ( e.g. order.create.success.) notificationId: 'not_123' ( the id of the configured notification that caused this event to be fired ) payload: json object, depends on the type. ( contained in the event ) }

Notification Events

You can receive notifications for the following events:

  • customer create success
  • customer create failure
  • customer update success
  • customer update failure
  • customer paymentInstrument add success
  • customer paymentInstrument add failure
  • customer paymentInstrument delete success
  • customer paymentInstrument delete failure
  • customer address create success
  • customer address create failure
  • customer address duplicate success
  • customer persona create success
  • customer persona create failure
  • customer persona duplicate success
  • order create success
  • order create failure
  • order update success
  • order update failure
  • transaction create success
  • transaction create failure
  • transaction update success
  • transaction update failure
  • paymentInstrument create success
  • paymentInstrument create failure
  • paymentInstrument expires
