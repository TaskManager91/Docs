Payment Page Integration
This feature allows you to create a customizable, standalone page for end customers and get informed about all payment events. The payment page is secure and simple to integrate.

Integrate and Use in 3 Steps

Depending on your needs, you can choose between a static and dynamic payment page.
Static Payment Page

1. Create your configuration in the Merchant Center
Fill all required fields (amount, currency, basket, etc.). See how to do it here.
2. Redirect end customer to the payment page
Generate an access link to your customized payment page (e.g. https://pp.payengine.de/checkouts/configuration_hiuzzvewrg). Check out all additional configuration options here and how to customize your payment page appearance here.
3. Get informed about payment status
Check the webhooks on all events you've subscribed for. See how to get notified here (Link to Notifications).
Dynamic Payment Page
1. Create your order
You should create an order on your server through our API. See how to do it (Link to Create Order).
2. Redirect end customer to the payment page
Generate access link to your customized payment page, which must contain existing configurationID and orderID (e.g. https://pp.payengine.de/checkouts/configuration_hiuzzvewrg?orderId=123). Check out all additional configuration options here and how to customize your payment page appearance here.
3. Get informed about payment status
Check the webhooks on all events you've subscribed for. See how to get notified here (Link to Notifications). Pull information for the order's status through our API. See example here.
Add Coupons to the Payment Page
Apply discounts for end customer as easy as it gets here.
Translate your Payment Page
Deliver multiple languages for better end customer experience. Check how to do this here.
View / Edit / Copy / Delete Payment Page configuration
Apply various actions on your configurations after creation. Explanation can be found here.


Create a Configuration

A configuration can be created in the Configurations sub-section by clicking on the “+” button. In the “Config” panel all required and optional parameters are ordered in selectable sections. Every configuration will get a system name, if a specific one is not provided.
To quickly create your payment page, you must fill only a few mandatory fields as follows:
! Important -> If "Allow dynamic configuration" is checked, values for these fields should be provided on creation of order (link to POST /orders). Note, that the value set in the order always supersedes the value set in the configuration for all fields.
  1. Payment methods (required) – checkbox selection of available payment methods, from which the end customer can choose from. “All” will select all payment methods. To create configuration at least one payment method should be selected.
  2. Payment Data (required) – input fields for amount and description, dropdown for currency and checkbox to capture the charge or create preauthorization (reserve the amount for later capture). Two of the fields are mandatory:
      • Amount – enter the desired number and use comma as separator for non-circular amount
      • Currency – select one of the available currencies
  3. Redirect (required) – this is a required section and it has three mandatory fields for transaction operation outcome:
      • Success URL – input field for the URL, where the customer will be redirected if transaction is successful
      • Error URL – input field for the URL, where the customer will be redirected if there is an error and transaction is not successful
      • Cancel URL – input field for the URL, where the customer will be redirected if transaction is cancelled
When filled in, just press “Save changes” button to create your configuration and to have ready to use Payment Page.

! Important -> If there are missing values for mandatory fields, error messages will appear and the configuration for new Payment Page won’t be saved.
